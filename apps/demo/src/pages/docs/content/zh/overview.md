# Yoga Canvas 是什么？

Yoga Canvas 是一个高性能、轻量级的 **Canvas 渲染引擎**，它将 **Yoga Flexbox** 的布局能力与 **Canvas 2D** 的绘制能力相结合，为 H5 和微信小程序提供了一套类 React 的声明式 UI 渲染方案。

---

## 1. 核心优势

- **🚀 卓越性能**: 基于 Canvas 绘制，即使在处理数千个节点的复杂列表或频繁交互时，依然能保持 60FPS 的丝滑体验。
- **🧩 声明式 UI**: 支持 JSX 和类 React 的组件化开发模式（View, Text, Image, ScrollView），上手成本极低。
- **📏 强大的布局系统**: 集成 Facebook 的 Yoga 引擎，完美支持 Flexbox 模型，包括伸缩、间距、绝对定位等。
- **🎨 丰富的样式支持**: 内置 Tailwind CSS 解析器，支持渐变、阴影、圆角裁剪、多行文本截断等 Web 级视觉能力。
- **📱 跨平台一致性**: 一套代码同时适配 H5 和微信小程序，通过平台适配层（Adapter）抹平环境差异。
- **⚡ 完整的交互系统**: 支持捕获与冒泡机制的事件分发系统，内置高精度的命中测试（Hit Testing）。

---

## 2. 适用场景

1. **高性能动态海报**: 告别传统的 DOM 转图片方案，直接在 Canvas 中实时渲染高保真、可交互的分享海报。
2. **可视化低代码平台**: 作为可视化编辑器的渲染内核，支持节点选中、高亮、拖拽、缩放和平移。
3. **复杂列表渲染**: 解决移动端 H5 在处理长列表、复杂卡片时的卡顿问题。
4. **游戏 UI 与轻量级引擎**: 为游戏开发提供一套标准的 UI 布局方案。

---

## 3. 核心概念

| 概念 | 说明 |
| :--- | :--- |
| **NodeDescriptor** | **声明式描述**。你通过 JSX 或 JSON 定义的“树”结构。 |
| **NodeTree / CanvasNode** | **运行时扁平树**。引擎内部用于布局计算、渲染绘制和命中测试的实体节点。 |
| **Yoga Engine** | **布局核心**。负责将 Flexbox 属性转化为每个节点的精确像素坐标。 |
| **Adapter** | **平台适配层**。负责抹平 H5 与小程序在图片加载、文本测量和 Canvas 导出方面的差异。 |

---

## 4. 为什么不直接用 DOM 或 WebGL？

- **对比 DOM**: DOM 在处理大规模动画或频繁交互时性能损耗巨大，且不支持直接导出高保真图片。Yoga Canvas 在内存占用和绘制效率上更胜一筹。
- **对比 WebGL / PixiJS**: WebGL 虽强，但包体积大且对文本、Flex 布局支持较弱。Yoga Canvas 专注于 UI 渲染，包体积极小，且布局能力完全对标 Web 标准。
